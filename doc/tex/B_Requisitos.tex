\section{Actores del sistema}

En esta sección se describen los distintos actores que interactúan con la aplicación web de detección de trazas. 
Los actores representan los perfiles de usuario y roles funcionales que acceden al sistema, diferenciándose en función de sus permisos y capacidades dentro de la aplicación. 
Esta distinción permite modelar de forma clara las responsabilidades y el alcance de cada tipo de usuario.

\subsection{Usuario Anónimo}

El \textbf{Usuario Anónimo} representa a cualquier persona que accede a la aplicación sin haber iniciado sesión ni disponer de una cuenta registrada. 
Este actor tiene acceso a las funcionalidades básicas del sistema, orientadas a la demostración y uso general de la herramienta de detección.

Concretamente, el Usuario Anónimo puede:
\begin{itemize}
	\item Cargar una imagen para su análisis.
	\item Ejecutar el proceso de detección y segmentación de trazas.
	\item Visualizar el resultado obtenido.
	\item Borrar la imagen cargada y reiniciar el flujo de procesamiento.
	\item Registrarse en el sistema.
\end{itemize}

Este perfil permite el uso inmediato de la aplicación sin barreras de acceso, manteniendo restringidas aquellas funcionalidades que requieren persistencia de datos o integración con servicios externos.

\subsection{Usuario Registrado}

El \textbf{Usuario Registrado} es un agente que dispone de una cuenta en el sistema y ha iniciado sesión correctamente. 
Este actor hereda todas las capacidades del Usuario Anónimo y, adicionalmente, puede acceder a funcionalidades avanzadas que requieren autenticación.

Además de las acciones disponibles para el Usuario Anónimo, el Usuario Registrado puede:
\begin{itemize}
	\item Iniciar y cerrar sesión en el sistema.
	\item Administrar su perfil de usuario.
	\item Descargar los resultados obtenidos tras el proceso de detección y segmentación.
	\item Acceder al módulo de integración GIS mediante Google Earth Engine.
	\item Definir una zona de interés y visualizar trazas dentro de dicha zona.
\end{itemize}

Este actor representa el perfil principal de uso de la aplicación, proporcionando acceso a funcionalidades de mayor valor añadido y a la integración con servicios externos.

\subsection{Administrador}

El \textbf{Administrador} es un usuario registrado con privilegios adicionales orientados a la gestión y mantenimiento del sistema. 
Este actor hereda todas las capacidades del Usuario Registrado y dispone de permisos específicos para la configuración interna de la aplicación.

Las funciones exclusivas del Administrador incluyen:
\begin{itemize}
	\item Administrar los modelos de detección utilizados por el sistema.
	\item Seleccionar el modelo activo que se empleará en el proceso de detección y segmentación.
\end{itemize}

La existencia de este rol permite separar claramente las tareas de uso del sistema de aquellas relacionadas con su configuración y control, facilitando la mantenibilidad y escalabilidad de la aplicación.
